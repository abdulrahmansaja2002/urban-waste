04

Modul Aplikasi & Fitur

Setiap modul merupakan domain fungsional yang terisolasi. Label MVP menandakan fitur wajib fase 1, V2 untuk fase 2, dan V3 untuk fase 3.

🚛

Modul Logistik & Penjemputan

Mengelola seluruh siklus request hingga selesainya penjemputan sampah.

  • Form request penjemputan oleh pelanggan (tipe sampah, berat estimasi, alamat, waktu) MVP
  • Dashboard driver: list pickup hari ini, navigasi ke Maps, konfirmasi tiba MVP
  • Input berat aktual + foto saat pickup, generate QR code material otomatis MVP
  • Status tracking real-time untuk pelanggan (dijadwalkan → dalam perjalanan → selesai) MVP
  • Geo-routing otomatis: optimasi urutan rute multi-pickup dalam satu hari V2
  • Integrasi Bluetooth timbangan digital IoT V2
  • AI sorting via kamera: deteksi otomatis jenis material V3
🏷️

Modul Material Tracking

Lacak perjalanan material dari limbah mentah hingga produk jadi menggunakan QR/Barcode.

  • Generate QR Code unik per batch material saat pickup selesai MVP
  • Update status material: diterima gudang → dalam produksi → selesai → terjual MVP
  • Halaman riwayat material untuk setiap produk (konsumen bisa scan QR produk) MVP
  • Laporan stok bahan baku real-time per tipe material MVP
  • Lot traceability: satu produk bisa berasal dari beberapa pickup batch V2
  • Blockchain anchoring untuk data integrity audit eksternal V3
⚙️

Modul Manajemen Produksi

Catat proses transformasi material menjadi produk jadi di lantai produksi.

  • Input produksi: ambil batch material → catat proses → output produk jadi + berat MVP
  • Manajemen kategori produk: furnitur, kompos, coffee pad, ornamen arsitektur MVP
  • Upload foto produk hasil produksi + spesifikasi teknis MVP
  • Dashboard stok produk jadi siap jual MVP
  • IoT sensor kompos: monitoring suhu + kelembapan realtime dengan alert V2
  • IoT kontrol mesin cetak ornamen arsitektur V3
🛍️

Modul Marketplace

Platform penjualan produk daur ulang dengan label transparansi material.

  • Galeri produk dengan filter kategori, material, harga MVP
  • Halaman detail produk: foto, spesifikasi, label "Riwayat Material" (link QR) MVP
  • Keranjang belanja + checkout (integrasi payment gateway Midtrans/Xendit) MVP
  • Manajemen pesanan: konfirmasi, pengiriman, selesai MVP
  • Review & rating produk oleh pembeli V2
  • B2B Portal: pemesanan custom volume besar untuk arsitek/desainer V2
  • API publik untuk integrasi platform e-commerce eksternal V3
📊

Modul Impact Dashboard

Laporan visual dampak lingkungan untuk pengguna, internal, dan laporan investor.

  • Dashboard pelanggan: total sampah disetorkan, produk terbuat dari material mereka MVP
  • Dashboard admin: total volume sampah, jumlah produk, revenue, top pelanggan MVP
  • Konversi visual: kg sampah → CO₂ offset, pohon setara, produk fungsional MVP
  • Export laporan PDF/CSV untuk keperluan reporting MVP
  • Impact certificate yang bisa dibagikan ke media sosial V2
  • Public impact page untuk laporan transparansi startup V2
👥

Modul Auth & User Management

Sistem autentikasi multi-role dan manajemen pengguna lintas platform.

  • Role-based: Customer, Driver, Operator Gudang, Admin, B2B Partner MVP
  • Registrasi + login email/password, OTP via WhatsApp/SMS MVP
  • Profil pengguna + riwayat transaksi (pickup & pembelian) MVP
  • Push notification: status pickup, pesanan, promo MVP
  • SSO Google / Apple Sign-In V2
  • Loyalty points & reward sistem untuk pelanggan aktif V3

FASE 01
MVP Core
Bulan 1–3
Target: sistem berjalan end-to-end dari pickup request hingga produk terdaftar di marketplace. Cukup untuk soft launch & demo investor.

Sprint 1–2 (Bulan 1): Foundation

  • Setup monorepo (Next.js, React Native, Fastify), konfigurasi PostgreSQL + Redis, CI/CD dasar
  • Implementasi auth: registrasi, login JWT, OTP WhatsApp (Fonnte/Wablas), RBAC 5 role
  • Modul profil pengguna dan manajemen alamat
  • Setup storage S3 (Cloudflare R2) untuk foto dan dokumen

Sprint 3–4 (Bulan 2): Pickup & Tracking

  • Customer App: form request pickup, tracking status real-time via WebSocket
  • Driver App: dashboard pickup hari ini, konfirmasi selesai, input berat + foto
  • Auto-generate QR Code unik per material batch saat pickup selesai
  • Admin panel: jadwal driver, monitoring semua pickup
  • Push notification: FCM untuk status update pickup

Sprint 5–6 (Bulan 3): Produksi & Marketplace

  • Operator panel: input produksi, kaitkan batch material, upload produk hasil
  • Marketplace web: listing produk, halaman detail + QR riwayat material
  • Integrasi Midtrans (payment gateway): transfer bank, QRIS, dompet digital
  • Impact dashboard basic: total kg, CO₂ offset, jumlah produk terbuat
  • QA + UAT internal, bug fixing, soft launch
FASE 02
Enhancement
Bulan 4–6
Target: tingkatkan pengalaman pengguna, tambah kapabilitas IoT awal, dan buka saluran B2B untuk arsitek & desainer.

Sprint 7–8 (Bulan 4): UX & Optimization

  • Geo-routing: integrasi Google Maps / OSRM untuk optimasi urutan rute driver multi-pickup
  • Review & rating produk oleh pembeli terverifikasi
  • SSO Google Sign-In untuk customer app
  • Impact certificate yang bisa dibagikan ke media sosial (PNG/PDF dinamis)

Sprint 9–10 (Bulan 5): IoT & Produksi

  • Integrasi timbangan digital via Bluetooth (protokol UART/BLE ke API mobile)
  • Setup MQTT broker (Mosquitto) + sensor kompos: monitoring suhu & kelembapan, alert threshold
  • Dashboard IoT realtime untuk operator produksi kompos
  • TimescaleDB untuk storage time-series data sensor

Sprint 11–12 (Bulan 6): B2B & Reporting

  • B2B Portal: form pemesanan custom volume besar, flow approval, invoice generator
  • Advanced reporting: laporan bulanan, tren material, konversi per kategori produk
  • Export laporan PDF otomatis untuk keperluan investor/CSR partner
  • Public impact page startup (transparansi data)
FASE 03
Scale & AI
Bulan 7+
Target: integrasi AI/ML untuk efisiensi operasional, ekspansi ekosistem dengan API publik, dan loyalty program pelanggan.

AI & Machine Learning

  • AI Sorting via kamera: model CV (YOLOv8) untuk klasifikasi sampah organik/anorganik di titik penjemputan
  • Demand forecasting: prediksi volume pickup berdasarkan histori untuk optimasi jadwal driver
  • Anomaly detection pada sensor IoT: deteksi mesin kompos bermasalah sebelum breakdown

Ecosystem & Scale

  • REST API publik untuk integrasi platform e-commerce eksternal (Tokopedia, Shopee)
  • Loyalty points system: reward pelanggan aktif, gamifikasi setoran sampah
  • Multi-tenant: support ekspansi ke kota lain dengan konfigurasi zona terpisah
  • Blockchain anchoring untuk sertifikasi data integrity produk daur ulang

Risiko & Strategi Mitigasi

Risiko Level Mitigasi
Kompleksitas IoT di awal
Integrasi hardware timbangan/sensor membutuhkan waktu debug panjang		 Tinggi Tunda IoT ke fase 2. MVP gunakan input manual untuk berat dan monitoring. Pilih hardware dengan SDK matang (ESP32 + library PubSubClient).
Data integrity material
QR code bisa disalahgunakan atau tidak konsisten bila proses manual		 Tinggi QR auto-generate hanya via sistem saat pickup selesai, tidak bisa dibuat manual. Setiap scan dicatat dengan timestamp + user yang melakukan scan.
Adopsi driver app rendah
Driver tidak mau / tidak bisa menggunakan app mobile		 Sedang Desain UI semaksimal mungkin sederhana, ukuran tombol besar. Sediakan alternatif WhatsApp bot untuk konfirmasi status sebagai fallback.
Payment gateway kompleks
Integrasi dan testing payment memakan waktu lama		 Sedang Gunakan Midtrans Snap (hosted payment page) di MVP — tidak perlu custom UI. Switch ke custom payment UI di fase 2.
Keamanan data pengguna
Data alamat, transaksi, dan kontak pelanggan rentan		 Sedang HTTPS wajib (Let's Encrypt), enkripsi password (bcrypt), JWT refresh token rotation, rate limiting API, audit log untuk aksi sensitif.
Scope creep
Fitur terus bertambah, MVP tidak pernah rilis		 Rendah Strict feature freeze setelah sprint planning. Semua fitur baru masuk backlog V2. Gunakan GitHub Issues + milestone untuk disiplin roadmap.
✅ Rekomendasi Tim MVP Minimal

Untuk membangun MVP dalam 3 bulan, tim ideal adalah: 1 Full-stack developer (Next.js + Node.js), 1 Mobile developer (React Native), 1 UI/UX designer (bisa part-time), dan 1 Product/QA. Total 3–4 orang. Satu developer berpengalaman yang menguasai full-stack + React Native bisa menjalankan sendiri dalam waktu lebih panjang (~5 bulan).

🚀 Prioritas Tertinggi untuk Launch

Dari semua modul, tiga hal yang paling krusial untuk bisa dipasarkan: (1) Pickup flow end-to-end yang reliable, (2) QR material traceability sebagai diferensiasi utama, dan (3) Impact dashboard yang visual dan shareable. Ketiga ini adalah nilai jual inti yang tidak dimiliki kompetitor.
